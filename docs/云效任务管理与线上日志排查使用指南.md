## 前置条件

### 1. 安装 AI 助手

```bash
# Claude Code（推荐）
npm install -g @anthropic-ai/claude-code

# 或 OpenAI Codex
npm install -g @openai/codex

# 或 Cursor（IDE 内置，无需额外安装）
```

### 2. 初始化项目配置

在你的项目根目录执行：

```bash
npx ai-engineering-init
```

选择需要初始化的工具（claude / codex / cursor / all），会自动安装技能文件和配置。

### 3. 获取必要的 Token

| Token | 用途 | 获取方式 |
|-------|------|---------|
| **云效个人访问令牌** | 云效 API 认证 | 云效 → 个人设置 → 个人访问令牌 → 创建（需「项目协作（读写）」权限） |
| **Grafana Service Account Token** | Loki 日志查询认证 | 每个 Grafana 环境独立创建：Administration → Service accounts → Add（Viewer 角色）→ Add token |

## 配置指南（按操作系统）

### 通用说明

配置涉及两个文件，均在项目根目录的 `.claude/` 下：

| 文件 | 用途 | 是否必须 |
|------|------|---------|
| `.claude/settings.json` | 云效 Token 配置 | 云效任务管理必须 |
| `.claude/skills/loki-log-query/environments.json` | Loki 多环境 Token 配置 | 线上日志查询必须 |
| `.claude/mysql-config.json` | MySQL 连接配置（可选） | 仅远程数据库查询需要 |

> **安全提示**：以上配置文件包含敏感 Token，请确保已加入 `.gitignore`，**不要提交到版本库**。

---

### macOS 配置

#### Step 1: 配置云效 Token

编辑 `.claude/settings.json`，在 `mcpServers` 中添加云效配置：

```json
{
  "permissions": {},
  "hooks": { ... },
  "mcpServers": {
    "yunxiao": {
      "type": "stdio",
      "command": "node",
      "args": ["-e", "// yunxiao token holder"],
      "env": {
        "YUNXIAO_ACCESS_TOKEN": "你的云效个人访问令牌"
      }
    }
  }
}
```

> **说明**：`yunxiao-task-management` 技能会从 `mcpServers.yunxiao.env.YUNXIAO_ACCESS_TOKEN` 读取 Token，通过 Node.js fetch 直接调用云效 Open API，不依赖 MCP 服务进程。`command` 和 `args` 为占位配置。

#### Step 2: 配置 Loki 日志查询 Token

从模板文件初始化配置：

```bash
cp .claude/skills/loki-log-query/environments.template.json \
   .claude/skills/loki-log-query/environments.json
```

然后编辑 `environments.json`，填入对应环境的 Token：

```json
{
  "active": "test13",
  "environments": {
    "test13": {
      "name": "测试13（主测试环境）",
      "url": "https://test13.xnzn.net/grafana",
      "token": "glsa_你的token值",
      "aliases": ["test13", "13"],
      "projects": ["test13", "test14", "test15"]
    },
    "monitor-dev": {
      "name": "Monitor 开发环境",
      "url": "https://monitor-dev.xnzn.net/grafana",
      "token": "glsa_你的token值",
      "aliases": ["monitor-dev", "mdev", "dev"],
      "projects": ["dev01", "dev02", "dev03"]
    }
  }
}
```

每个 Grafana 环境需要独立创建 Token：
1. 登录对应 Grafana URL
2. 左侧菜单 → **Administration** → **Service accounts**
3. **Add service account**（名称：`claude-loki-reader`），角色选 **Viewer**
4. 点击 **Add service account token** → 复制 Token
5. 粘贴到对应环境的 `token` 字段

#### Step 3: 配置 MySQL（可选）

本地开发环境可零配置——技能会自动从项目的 `bootstrap-dev.yml` 提取连接信息。

如需连接远程数据库，创建 `.claude/mysql-config.json`：

```json
{
  "environments": {
    "local": { "host": "127.0.0.1", "port": 3306, "user": "root", "password": "你的密码" },
    "dev":   { "host": "dev-db.example.com", "port": 3306, "user": "dev_user", "password": "xxx" }
  },
  "default": "local"
}
```

确保 MySQL 客户端已安装：

```bash
# macOS（Homebrew）
brew install mysql-client

# 验证
mysql --version
# 或者
/opt/homebrew/opt/mysql-client/bin/mysql --version
```

#### Step 4: 验证配置

```bash
# 启动 Claude Code
claude

# 测试云效连接
> "查看我的待办任务"

# 测试 Loki 连接
> "查 test13 的 ERROR 日志"

# 测试 MySQL 连接（如已配置）
> "查一下本地数据库的表结构"
```

### Windows 配置

#### Step 1: 安装 Node.js

确保已安装 Node.js >= 16：

```powershell
# 下载安装：https://nodejs.org/
# 或使用 winget
winget install OpenJS.NodeJS.LTS

# 验证
node --version
npm --version
```

#### Step 2: 安装 AI 助手

```powershell
npm install -g @anthropic-ai/claude-code
```

> **注意**：Windows 下 Claude Code 需要在 **Windows Terminal** 或 **VS Code Terminal** 中运行，不支持传统 CMD。

#### Step 3: 初始化项目

```powershell
cd 你的项目目录
npx ai-engineering-init
```

#### Step 4: 配置 Token

与 macOS 相同，编辑以下文件（使用 VS Code 或任意文本编辑器）：

- `.claude/settings.json` — 添加云效 Token
- `.claude/skills/loki-log-query/environments.json` — 添加 Loki Token

> **Windows 路径注意**：配置文件中的路径使用正斜杠 `/` 或双反斜杠 `\\`。

#### Step 5: 安装 MySQL 客户端（可选）

```powershell

# 方式 3：使用 Chocolatey
choco install mysql

# 验证
mysql --version
```

确保 `mysql` 命令在 PATH 中可用。如果不在 PATH 中，需要手动添加：

```powershell
# 查找 mysql.exe 位置（通常在 C:\Program Files\MySQL\MySQL Server 8.x\bin\）
# 添加到系统 PATH 环境变量
```

#### Step 6: 验证


#### 实操示例

```
# Step 1: 收到 Bug 报告，贴日志
你: "线上报错了，日志如下：
2026-03-07 09:16:53.039,a53dd0b0cc62bf4a79a63e77444f6f3f,553722740746489856,,
ERROR 1 --- [XNIO-1 task-3] n.x.f.c.request.RequestLoggingFilter : 47 :
NullPointerException at OrderInfoService.java:128"

# Step 2: AI 自动分析
AI: （自动激活 bug-detective）
    → 分析错误类型：NullPointerException
    → 定位文件：OrderInfoService.java:128
    → 评估是否需要查库：需要（特定租户数据问题）
    → 提取租户 ID/数据库名：553722740746489856

# Step 3: 查线上 Loki 日志（获取完整链路）
你: "用 traceId a53dd0b0cc62bf4a79a63e77444f6f3f 查完整日志"
AI: （自动激活 loki-log-query）
    → 读取 environments.json，使用 active 环境
    → 调用 Grafana Loki API 查询
    → 返回完整请求链路日志（请求 → 处理 → 响应）

# 或者指定环境
你: "去 monitor-dev 查一下这个 traceId 的日志"
AI: → 切换到 monitor-dev 环境查询

# Step 4: 查数据库验证数据
你: "查一下 553722740746489856 库的 order_info 表，看看 id=xxx 的数据"
AI: （自动激活 mysql-debug）
    → 连接数据库（自动从 bootstrap-dev.yml 提取连接信息）
    → 执行 SELECT（只读查询，安全规范强制执行）
    → 返回数据状态

# Step 5: 修复
AI: → 根据日志 + 数据分析，给出修复方案
    → 修改代码

# Step 6: 更新云效任务
你: "把对应的 Bug 单改为已修复"
AI: → 调用云效 API 更新状态
```

1. 先按接口路径搜索，提取 traceId

### MySQL 数据库查询 (mysql-debug)

#### 零配置使用（本地开发）

如果项目中有 `bootstrap-dev.yml`，技能会自动提取连接信息，无需额外配置：

```
你: "查一下 546198574447230976 库的 order_info 表"
你: "SELECT * FROM order_info WHERE id = 123 AND del_flag = 2"
```

#### 指定环境

```
你: "连 dev 环境查一下 order_info"
你: "用生产库看看"
```

#### 从日志自动提取数据库名

日志格式中第三段数字就是租户 ID（= 数据库名）：

```
2026-03-07 09:16:53.039,-,546198574447230976,, ERROR ...
                           ↑ 数据库名
```

AI 会自动提取并使用。

#### 安全规范

- **只允许 SELECT / DESCRIBE / SHOW / EXPLAIN**（只读）
- 自动添加 `LIMIT 50` 防止全表扫描
- 敏感字段（密码、手机号、身份证）自动脱敏
- **禁止** INSERT / UPDATE / DELETE 等写操作

## 常见问题 FAQ

### Q1: 云效 Token 怎么获取？

1. 登录 [云效](https://devops.aliyun.com/)
2. 右上角头像 → **个人设置**
3. 左侧 **个人访问令牌**
4. **创建令牌**，选择权限：
   - `项目协作（读取）` — 查询任务、搜索项目
   - `项目协作（读写）` — 修改状态、创建提测单、评论、工时

### Q2: Grafana Token 过期了怎么办？

重新创建 Service Account Token：

1. 登录对应 Grafana 环境
2. Administration → Service accounts
3. 找到 `claude-loki-reader` → Add service account token
4. 复制新 Token 到 `environments.json`

### Q3: Loki 查询返回 "invalid API key" 怎么办？

说明对应环境的 Token 为空或已过期。检查 `.claude/skills/loki-log-query/environments.json` 中对应环境的 `token` 字段是否已填写。

### Q4: MySQL 连接失败怎么办？

检查三级降级来源：

1. **bootstrap-dev.yml 存在吗？** — 技能会从 `spring.dataset.system.master` 节点提取连接信息
2. **mysql 客户端安装了吗？** — 运行 `which mysql` 或 `mysql --version` 检查
3. **密码正确吗？** — 如果 bootstrap-dev.yml 中的密码不对，创建 `.claude/mysql-config.json` 手动指定

### Q5: 可以同时查多个环境的日志吗？

目前需要一个一个环境查。可以通过切换环境来查不同的 Grafana：

```
你: "先查 test13 的日志"
你: "再去 monitor-dev 查一下同样的 traceId"
```

### Q6: 我不在 leniu 项目，能用这些技能吗？

- `yunxiao-task-management`：只要你有云效账号和 Token，任何项目都可以用
- `loki-log-query`：需要你的 Grafana 环境支持 Loki，修改 `environments.json` 中的 URL 和 Token
- `mysql-debug`：通用技能，配置好数据库连接即可
- `bug-detective`：通用 Bug 排查方法论，但部分错误模式库是针对 leniu 项目的

### Q7: Windows 下 curl 命令不可用怎么办？

Loki 查询技能使用 Node.js `fetch`（而非 curl）来调用 API，不依赖系统 curl。确保 Node.js >= 18（内置 fetch）即可。

如果 Node.js < 18，升级 Node.js：

```powershell
winget install OpenJS.NodeJS.LTS
```

### Q8: 配置文件会被提交到 Git 吗？

不会。以下文件应在 `.gitignore` 中：

```gitignore
# AI 工具敏感配置
.claude/settings.json
.claude/skills/loki-log-query/environments.json
.claude/mysql-config.json
```

如果你的项目使用了 `npx ai-engineering-init`，这些排除规则已自动处理。

## 附录：配置文件模板

### .claude/settings.json（云效 Token）

```json
{
  "permissions": {},
  "hooks": {},
  "mcpServers": {
    "yunxiao": {
      "type": "stdio",
      "command": "node",
      "args": ["-e", "// yunxiao token holder"],
      "env": {
        "YUNXIAO_ACCESS_TOKEN": "在此填写你的云效个人访问令牌"
      }
    }
  }
}
```

### .claude/skills/loki-log-query/environments.json（Loki Token）

> 项目已提供模板文件 `environments.template.json`，复制后填入 Token 即可：
> ```bash
> cp .claude/skills/loki-log-query/environments.template.json \
>    .claude/skills/loki-log-query/environments.json
> ```

```json
{
  "active": "test13",
  "environments": {
    "test13": {
      "name": "测试13（主测试环境）",
      "url": "https://test13.xnzn.net/grafana",
      "token": "glsa_在此填写Token",
      "aliases": ["test13", "13"],
      "projects": ["test13", "test14", "test15"]
    },
    "monitor-dev": {
      "name": "Monitor 开发环境",
      "url": "https://monitor-dev.xnzn.net/grafana",
      "token": "glsa_在此填写Token",
      "aliases": ["monitor-dev", "mdev", "dev"],
      "projects": ["dev01", "dev02", "dev03"]
    }
  }
}
```
